What's New in 2.3 (still version 2.2)

Reminders are smarter
- New "Notify" choice on every reminder: Everyone, Admins only, or a specific person.
- Reminders assigned to a person now appear in that person's My To-Do alongside their chores.
- Admin-only reminders are hidden from non-admins entirely.
- The Reminders tab now only holds loose "for the family" notes; targeted reminders live with the assignee.

My To-Do improvements
- Bell button in the quick-add bar -- type a line, tap the bell to turn it into a real reminder with notify target, date, repeat, color, etc.
- Tap a Recently Done item to un-complete it or open the editor (no more accidentally lost progress).
- Stacked avatars on admin/everyone reminders so you can see who it's for at a glance.

Chore bundles can now be edited
- Tap an expanded bundle's new Edit button to change name, category, assignee, or bonus points.
- Changing the bundle's category cascades to every child chore so they stay consistent.

Theme + UI polish
- New Sunrise theme: dawn blue paired with sunrise coral.
- Family List hero matches its orange dashboard tile.
- Saved Locations actually wired into reminder/event/trip pickers -- pick a stored place without searching.
- Weekday repeat: Mon-Fri quick pick, plus a multi-select weekday chip row.
- Custom repeat picker now reachable from the New Task screen.

Reliability fixes from 2.2
- CloudKit env split: Debug talks to Development, Release talks to Production. 2.0 / 2.1 were silently using Dev.
- 2-second debounce on remote change pipeline so SQLite WAL doesn't block CloudKit exports.
- Kid mode no longer gets stuck after role changes.

What to Test
- Create reminders targeting Everyone / Admins only / a specific person. Verify only the right people see them in My To-Do.
- Edit a chore bundle's category. Confirm child chores follow.
- Pick the new Sunrise theme in Settings -> Appearance.
- Try the quick-add bell -- it should open a fresh reminder editor pre-filled with what you typed.
